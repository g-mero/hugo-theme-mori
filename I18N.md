# i18n Internationalization Support

Mori theme adopts a **minimal translation** i18n design philosophy.

## Design Philosophy

### Core Principle: Minimize Translation Requirements

- âœ… **User-controlled content** - Articles and page titles are translated by users themselves
- âœ… **Menu uses identifiers** - Automatically translates menu items via identifiers
- âœ… **Minimal UI text** - Only translates necessary interface text
- âŒ **No user content translation** - Theme does not translate user articles and pages

### What Needs Translation?

Only translate these UI elements:

- Menu items (via identifier)
- Page section titles (like "Recent Posts")
- Copyright and other fixed text

### What Does NOT Need Translation?

- Article titles and content (users create multilingual versions themselves)
- Page titles (defined in front matter by users)
- Custom text

### 2. Menu Configuration

**Recommended Approach: Use Separate Menu Configuration Files**

Create `config/_default/menus.en.toml`:

```toml
[[main]]
identifier = 'home'      # Use identifier, theme auto-translates
pageRef = '/'
weight = 10

[[main]]
identifier = 'about'
pageRef = '/about'
weight = 20

[[main]]
identifier = 'posts'
pageRef = '/posts'
weight = 30
```

Create `config/_default/menus.zh-cn.toml`:

```toml
[[main]]
identifier = 'home'      # Same identifier
pageRef = '/'
weight = 10

[[main]]
identifier = 'about'
pageRef = '/about'
weight = 20

[[main]]
identifier = 'posts'
pageRef = '/posts'
weight = 30
```

**How it Works:**

- Use `identifier` instead of `name`
- Theme automatically looks up translation from `i18n/`
- `identifier = 'home'` â†’ Shows "Home" in English, "é¦–é¡µ" in Chinese

**Custom Menu Names:**

If you need custom names, specify `name` directly:

```toml
[[main]]
identifier = 'custom'
name = 'My Custom Link'    # Fixed name, not translated
url = 'https://example.com'
weight = 40
```

3. âœ… **Keep it minimal** - Only translate necessary UI text
